# ミツクル デザインシステム

ミツクルは「見積りと制作依頼をワンストップで」提供するWEBマーケティング支援サービス。
サイトを実装・改修するときは、以下のルールに従ってデザインの一貫性と品質を保つ。

## 0. デザインの基本方針

- **グリーン基調**。既存ブランドの親しみやすいグリーンを主役に据える。
- **モダン × シンプル × 分かりやすい**。装飾を足すより、余白と整理で伝える。
- 参考にする世界観: Notta のような SaaS 系LPの「広い余白・明快なセクション構成・大きく分かりやすいCTA」。
  ただし配色はブルーではなくミツクルのグリーンに置き換える。
- 迷ったら「足す」より「引く」。要素を減らし、1セクション1メッセージにする。

## 1. カラーパレット

CSS変数として定義し、ハードコードした色は使わない。（グリーンはスクショからの推定値。実CSSがあれば差し替え可）

```css
:root {
  --color-primary:       #7CB342;  /* ミツクルのグリーン（主役・CTA・強調文字） */
  --color-primary-dark:  #689A38;  /* ボタンhover・押下 */
  --color-primary-light: #EFF6E4;  /* セクション背景・淡いブロック */
  --color-text:          #2B2B2B;  /* 本文 */
  --color-text-muted:    #6B7280;  /* 補足テキスト */
  --color-bg:            #FFFFFF;  /* 背景 */
  --color-bg-subtle:     #F7F8F5;  /* セクション交互背景 */
  --color-border:        #E5E7EB;  /* 罫線・カード枠 */
}
```

- 強調したい単語（見出し内のキーワード等）はグリーンにする。ミツクルらしさが出る。
- グリーンは「CTA・キーワード・アイコン」に絞る。広い面積のベタ塗りは避け、白を基調にする（Notta的な軽さ）。
- アクセントを増やしたくなったら、まず余白とサイズで階層を作れないか検討する。

## 2. タイポグラフィ

```css
:root {
  --font-base: 'Noto Sans JP', sans-serif;
}
```

| 用途 | サイズ(PC) | 太さ | 行間 |
|------|-----------|------|------|
| H1（ヒーロー見出し） | 36–44px | 700 | 1.35 |
| H2（セクション見出し） | 26–30px | 700 | 1.4 |
| H3 | 18–20px | 600 | 1.5 |
| 本文 | 15–16px | 400 | 1.8 |
| 補足 | 13px | 400 | 1.6 |

- 日本語本文の行間は広め（1.8前後）で可読性を確保。
- セクション見出しは中央寄せが基本（Notta風の整然とした構成）。
- 見出しの一部キーワードだけグリーンにして視線を誘導する。
- 本文コンテナは max-width 720px 程度に収め、長すぎる行を作らない。

## 3. レイアウト規約

- コンテンツ最大幅: `--container: 1120px;`
- **セクション上下余白は広めに**: PC 96–120px / モバイル 56–72px（Notta的な余白感の肝）
- 余白は 8px グリッド（8 / 16 / 24 / 32 / 48 / 64 / 96…）で統一。
- セクションは白 ↔ `--color-bg-subtle` を交互にして区切る（罫線に頼りすぎない）。
- 装飾の淡いグリーンのブロブ（既存ヒーローの背景）は使ってよいが、控えめに。

## 4. コンポーネントの作法

### ボタン（角丸）
- **角丸を効かせる**: `border-radius: 9999px;`（ピル型）を基本。最低でも 8px。
- プライマリ: 背景 `--color-primary` / 文字白 / hover で `--color-primary-dark`。
- セカンダリ: 枠線 `--color-primary` / 文字グリーン / 背景透明。
- パディングは大きめ（例: 14px 28px）。CTAは画面内で目立つサイズにする。
- 最小タップ領域 44px 以上（モバイル）。

### カード（特徴・サービス紹介）
- 白背景 + 淡い影 **または** 1pxの淡い枠線、どちらか一方のみ。
- 角丸 12–16px。内側パディング 24–32px。
- アイコンは円形の淡いグリーン背景（`--color-primary-light`）に乗せると既存トンマナに合う。
- 3カラムグリッド基準。モバイルで1カラムに落とす。

### セクション見出しブロック
- 中央寄せの大見出し + 1行リード文（`--color-text-muted`）のセット。
- Nottaのように「見出し→説明→カード群→CTA」の順で1セクションを構成する。

### フォーム
- ラベルは入力欄の上。フォーカス時はグリーンのアウトライン。
- 角丸 8px で統一。

## 5. ページ構成のテンプレート（Notta参考）

ランディング系ページは概ねこの並びを基本形にする:

1. ヘッダー（ロゴ + ナビ + CTAボタン）
2. ヒーロー（キーワードをグリーン強調した大見出し + リード + CTA + イラスト/画像）
3. 信頼の証（導入実績・ロゴ・数値などがあれば）
4. お悩み/価値提案（カード3つ程度）
5. 機能・できること（アイコン付きカードグリッド）
6. 選ばれる理由（番号付きで2〜4点）
7. 料金（必要なら。カード比較）
8. よくある質問（アコーディオン）
9. 末尾CTA（再度大きく）
10. フッター

## 6. コーディング規約

- **モバイルファースト**。ブレークポイントは 768px / 1024px を基準。
- セマンティックHTML（header / nav / main / section / article / footer）。
- 画像は `loading="lazy"` と適切な alt。
- 見出し階層を崩さない（h1は1ページ1つ）。SEO内部対策を意識。
- アクセシビリティ: コントラスト比 4.5:1 以上、フォーカス可視化を消さない。

## 7. やってはいけないこと（アンチパターン）

- グリーンを広い面積にベタ塗りして重くする（白基調＋グリーン差し色が正解）。
- ボタンを角張らせる（必ず角丸／ピル型）。
- 余白を詰めすぎてNotta的な「軽さ・分かりやすさ」を失う。
- フォントを3種類以上混在させる。
- 余白をグリッド外の中途半端な値にする（例: 13px, 27px）。
- 色や余白をインラインでベタ書きする（必ずCSS変数）。
